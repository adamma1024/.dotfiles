# /setup-workspace — Bootstrap Claude Code for Prometheus Projects

Set up a complete Claude Code environment for any project generated by Prometheus Incubator. This creates the CLAUDE.md, hooks, settings, and skills tailored to the project's tech stack.

## What to do

### Step 1: Detect Project Type
Read `package.json` (or equivalent) to determine:
- Framework: Next.js, Vite, Express, SwiftUI, Jetpack Compose, etc.
- Language: TypeScript, JavaScript, Python, Swift, Kotlin, Rust
- Package manager: pnpm, npm, yarn, bun
- Test runner: vitest, jest, pytest, XCTest
- Linter: eslint, biome, swiftlint
- Database: Supabase, Neon, Prisma, Drizzle

### Step 2: Generate CLAUDE.md
Create a `CLAUDE.md` in the project root with this structure:

```markdown
# [Project Name]

## Quick Facts
- **Stack**: [detected from package.json]
- **Package Manager**: [pnpm|npm|yarn]
- **Dev Command**: `[pm] dev`
- **Build Command**: `[pm] build`
- **Test Command**: `[pm] test`
- **Lint Command**: `[pm] lint`

## Key Directories
[Auto-detect from src/ structure]

## Code Style
- TypeScript strict mode
- Prefer `interface` over `type` (except unions/intersections)
- No `any` — use `unknown`
- Early returns, avoid nested conditionals

## Git Conventions
- Branch: `{initials}/{description}`
- Commits: Conventional Commits (`feat:`, `fix:`, `docs:`)
- Never commit .env, credentials, or secrets

## Critical Rules

### Error Handling
- Never swallow errors silently
- Always show user feedback for errors
- Log errors for debugging

### UI States (if frontend)
- Handle: loading, error, empty, success
- Show loading ONLY when no data exists
- Every list needs an empty state

### Testing
- Test-first: write failing test before implementation
- One behavior per test
- Factory pattern for mock data: `getMockX(overrides)`
```

### Step 3: Generate .claude/settings.json (project-level)
Create `.claude/settings.json` with:

```json
{
  "permissions": {
    "allow": [
      "Bash([pm] test:*)",
      "Bash([pm] lint:*)",
      "Bash([pm] build:*)",
      "Bash([pm] dev:*)",
      "Bash(git status:*)",
      "Bash(git diff:*)",
      "Bash(git log:*)",
      "Bash(git add:*)",
      "Bash(git commit:*)",
      "Bash(lsof:*)",
      "Bash(curl:*)",
      "Bash(docker:*)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "if [[ \"$CLAUDE_TOOL_INPUT_FILE_PATH\" =~ \\.(ts|tsx|js|jsx)$ ]]; then npx prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\" 2>/dev/null && echo '{\"suppressOutput\": true}'; fi",
            "timeout": 15
          }
        ]
      }
    ]
  }
}
```

Replace `[pm]` with detected package manager.

### Step 4: Generate .claude/commands/
Create these project-level commands:

**`.claude/commands/dev.md`**:
```
Start the development server. Run `[pm] dev` and report the URL when ready.
```

**`.claude/commands/check.md`**:
```
Run all quality checks in parallel:
1. `[pm] lint` — fix any lint errors
2. `[pm] test` — all tests must pass
3. `npx tsc --noEmit` — no type errors
Report results as a table.
```

**`.claude/commands/deploy.md`**:
```
Pre-deploy checklist:
1. Run /check to verify quality
2. `[pm] build` — must succeed
3. `git status` — no uncommitted changes
4. Show the deploy command for the configured platform
```

### Step 5: Print Summary
Show the user what was created:

```

Created:
  CLAUDE.md          — Project context and rules
  .claude/settings.json — Permissions and hooks
  .claude/commands/dev.md — Start dev server
  .claude/commands/check.md — Quality checks
  .claude/commands/deploy.md — Deploy checklist

Detected:
  Framework: [X]
  Language: [X]
  Package Manager: [X]
  Test Runner: [X]

Try:
  /dev    — Start development
  /check  — Run quality checks
  /grill  — Find flaws in your code
```

## Rules
- Auto-detect everything from existing files — don't ask the user
- If something can't be detected, use sensible defaults
- Never overwrite existing CLAUDE.md — merge or ask first
- Keep CLAUDE.md under 80 lines — every line must earn its place
- Include only rules that prevent mistakes, not obvious conventions
